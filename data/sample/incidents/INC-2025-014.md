# INC-2025-014: Auth outage from expired JWKS cache

severity: high
affected services: auth-svc, api-gateway
date: 2025-11-03

## Summary
A 22-minute outage where all authenticated requests returned 401. Root cause was
the api-gateway caching the JWKS for 24h while auth-svc rotated signing keys early
due to an unrelated key-compromise drill. The gateway kept validating against the
retired public key.

## Resolution
Reduced gateway JWKS cache TTL to 10 minutes and added a forced refresh on
consecutive signature-validation failures. Added an alert when the active signing
key id is absent from the gateway's cached JWKS.

## Action items
- Keep the previous public key in JWKS for two rotation windows, not one.
- Add a synthetic probe that validates a freshly issued token end-to-end.
